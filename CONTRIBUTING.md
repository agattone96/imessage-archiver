# Contributing to Archiver

Thank you for your interest in contributing to Archiver! We welcome contributions to help improve this tool.

## How to Contribute

### Reporting Bugs
- Use the **Bug Report** template when opening an issue.
- Provide a clear description of the issue and steps to reproduce.
- Include environment details (macOS version, etc.).

### Feature Requests
- Open an issue to discuss new features before starting development.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bugfix.
3. Ensure your code follows the existing style.
4. Submit a pull request with a clear description of changes.

## Development Setup
1. Clone the repository.
2. Run `./build_electron.sh` to install dependencies and build the app.
3. Use `npm start` to run the application in development mode.

## Code of Conduct
Please be respectful and professional in all interactions within the project.
